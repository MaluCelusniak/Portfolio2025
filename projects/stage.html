Stage - Ubilink

Développement d’outils internes et externes

Description : Objectif principal de développer et d’adapter plusieurs outils numériques destinés à être utilisés à la fois en interne (par les collaborateurs) et en externe (par les clients). Ces outils devaient s'intégrer à des environnements déjà en place : d’une part le site CRM utilisé par les commerciaux (SuiteCRM), d’autre part la plateforme de back-office de l’entreprise.

J’ai effectué mon stage de deuxième année au sein de la société Ubilink, spécialisée dans les solutions numériques 3D. Ce stage avait pour objectif principal de développer et d’adapter plusieurs outils numériques destinés à être utilisés à la fois en interne (par les collaborateurs) et en externe (par les clients). Ces outils devaient s'intégrer à des environnements déjà en place : d’une part le site CRM utilisé par les commerciaux (SuiteCRM), d’autre part la plateforme de back-office de l’entreprise.


Parmi les différents projets réalisés, trois m’ont particulièrement marqué par leur richesse technique et leur diversité :

Outil d’inventaire – "Inventory"

Compétences : PHP, JavaScript, JSON, système multi-langue, responsive web design

L’outil Inventory permet la gestion d’éléments virtuels à placer dans des environnements 3D. Mon travail a consisté à modifier la page de gestion des objets, notamment pour permettre leur édition multilingue (français/anglais). Cette tâche impliquait de manipuler des fichiers content.json, qui contiennent les métadonnées des objets. Il fallait assurer la correspondance parfaite entre les champs en français et en anglais, sans altérer la structure du JSON.

Ce projet m’a demandé de :

  • - Maîtriser le JSON.parse() pour gérer des données complexes,
  • - Introduction à l’utilisation de SVG dans l’UI,
  • - Compréhension d’un code legacy et adaptation à une architecture existante,

Cette mission m’a confronté à une problématique concrète de synchronisation entre les langues dans une interface web, à la gestion fine de fichiers de configuration, et au respect d’un système de formats déjà utilisé par d’autres équipes. Elle m’a également permis de mieux comprendre comment intégrer des interactions SVG dans une interface responsive.

Outil de visioconférence

Compétences : WebRTC, Node.js, TURN/STUN servers, architecture serveur

Un autre projet du stage concernait la mise en place d’un système de visioconférence. L’objectif était d’étudier différentes solutions open source comme Jitsi ou Hello, d’en comprendre les architectures (serveurs TURN/STUN, WebRTC), puis d’implémenter une première version de l’interface web permettant la création et la transmission de liens d’appel sécurisés.

Après étude, nous avons développé une interface d’entrée où les utilisateurs peuvent générer un lien personnalisé d’appel à partager. Cette interface reposait sur une clé générée et transmise via URL pour sécuriser l’accès. J’ai réalisé l’étude technique, conçu cette interface, puis passé le relais à un autre stagiaire pour l’intégration finale. Ce projet m’a permis de mieux comprendre les enjeux liés à la communication en temps réel sur le web, et les défis techniques posés par WebRTC.

Ce projet m’a demandé :

  • - Maîtriser le JSON.parse() pour gérer des données complexes,
  • - Introduction à l’utilisation de SVG dans l’UI,
  • - Compréhension d’un code legacy et adaptation à une architecture existante,

Cette mission m’a confronté à une problématique concrète de synchronisation entre les langues dans une interface web, à la gestion fine de fichiers de configuration, et au respect d’un système de formats déjà utilisé par d’autres équipes. Elle m’a également permis de mieux comprendre comment intégrer des interactions SVG dans une interface responsive.

Calendrier de prise de rendez-vous intégré au CRM

Compétences : PHP, SQL, JavaScript, gestion des fuseaux horaires, sécurité web, manipulation des dates, intégration CRM

Enfin, le projet le plus complet de mon stage a été le développement d’un système de prise de rendez-vous en ligne connecté au CRM. Ce module devait permettre à des clients externes de réserver un créneau pour un appel téléphonique ou une visioconférence, tout en s'intégrant parfaitement au système de gestion de leads du CRM SuiteCRM.

Ce projet m’a demandé de :

  • - développer un système de génération de créneaux disponibles (en PHP) en tenant compte des indisponibilités déjà enregistrées,
  • - gérer la logique de fuseau horaire entre l’utilisateur, le serveur (GMT 0) et le CRM (affichage correct),
  • - sécuriser les requêtes côté serveur (sessions, captcha),
  • - insérer correctement les rendez-vous dans les différentes tables du CRM (leads, meetings,meetings_users),
  • - envoyer un mail de confirmation avec un lien d’annulation sécurisé.

Ce projet m’a beaucoup appris sur les interactions complexes entre plusieurs bases de données, les limites des API internes (notamment les beans de SuiteCRM qui nécessitent une authentification), et sur l’importance de sécuriser les communications serveur/client dans un contexte d’exposition web. J’ai également dû prendre en compte des problématiques d’usage réel : comment empêcher un robot de prendre tous les créneaux ? Comment afficher dynamiquement les créneaux valides en fonction de deux commerciaux différents ?
